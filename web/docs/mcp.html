What is MCP?

Model Context Protocol (MCP) is the emerging standard for connecting AI agents to external tools via a stdio JSON-RPC 2.0 transport. Solace Browser implements an MCP server that exposes 7 browser tools to any MCP-capable agent: navigate, click, fill, screenshot, snapshot, evaluate, and aria_snapshot.

This means your AI coding agent can browse the web, click buttons, fill forms, take screenshots, and capture evidence — all through Solace Browser's approval gates. Every action still requires your approval.

Setup (30 seconds)

Make sure Solace Browser is running first:

python3 solace_browser_server.py --port 9222 &
python3 web/server.py --port 8791 &

Then start the MCP server:

python3 mcp_server.py

Or use npx:

npx solace-browser --mcp

Claude Code

Add this to your CLAUDE.md or project settings:

{
  "mcpServers": {
    "solace": {
      "command": "python3",
      "args": ["/path/to/solace-browser/mcp_server.py"]
    }
  }
}

Or using npx (no path needed):

{
  "mcpServers": {
    "solace": {
      "command": "npx",
      "args": ["solace-browser", "--mcp"]
    }
  }
}

After adding the config, Claude Code will show solace as an available MCP server. Ask Claude to navigate to a URL or take a screenshot and it will use Solace Browser automatically.

Cursor / Codex

Cursor and Codex support MCP servers via their tools configuration. Add the same JSON block to your agent's tools settings:

# Cursor: .cursor/mcp.json
{
  "mcpServers": {
    "solace": {
      "command": "python3",
      "args": ["/path/to/solace-browser/mcp_server.py"]
    }
  }
}

The 7 browser tools

navigate

Navigate the browser to a URL. Returns page title, final URL, and HTTP status.

// Input
{ "url": "https://gmail.com", "wait_until": "load" }

// Output
{ "url": "https://mail.google.com/...", "title": "Inbox - Gmail", "status": 200 }

click

Click a DOM element by CSS selector or ARIA role+name.

// Input
{ "selector": "button[aria-label='Compose']", "timeout_ms": 5000 }

// Output
{ "clicked": true, "element_text": "Compose", "elapsed_ms": 340 }

fill

Fill a form field with text. Supports CSS selectors and label-based targeting.

// Input
{ "selector": "input[name='subject']", "value": "Hello from Solace" }

// Output
{ "filled": true, "selector": "input[name='subject']" }

screenshot

Take a screenshot of the current page or a specific element. Returns base64-encoded PNG.

// Input
{ "selector": ".inbox-content", "full_page": false }

// Output
{ "format": "png", "data": "<base64>", "width": 1280, "height": 720 }

snapshot

Capture a full-page HTML snapshot. Returns the cleaned HTML content.

// Input
{ "include_scripts": false }

// Output
{ "html": "<!DOCTYPE html>...", "url": "https://...", "title": "..." }

evaluate

Execute JavaScript in the browser page context. Returns the result.

// Input
{ "expression": "document.querySelectorAll('.email').length" }

// Output
{ "result": 47, "type": "number" }

aria_snapshot

Return the ARIA accessibility tree of the page. Ideal for agents that need to understand page structure without parsing HTML.

// Input
{}

// Output
{ "tree": [{ "role": "main", "name": "Inbox", "children": [...] }] }

Usage examples

Example: Ask Claude Code to take a screenshot

// In your Claude Code chat:
"Take a screenshot of https://gmail.com and tell me what you see"

// Claude will call:
solace.navigate({ url: "https://gmail.com" })
solace.screenshot({})
// Then describe what it sees in the screenshot

Example: Automated web research

// Claude Code navigates + extracts data:
solace.navigate({ url: "https://news.ycombinator.com" })
solace.evaluate({ expression: "Array.from(document.querySelectorAll('.athing .title a')).slice(0,10).map(a=>a.textContent)" })
// Returns: ["Story 1", "Story 2", ...]

OAuth3 + budget gates

Even when called by an agent via MCP, every sensitive action requires your approval. The OAuth3 confirmation dialog appears in the Solace Browser UI. You approve or reject in the UI; the agent waits for your decision.

Budget gates also apply: if an agent tries to send more emails than your daily budget allows, the action is blocked and the agent receives a budget error — not a silent failure.

  • Navigate / screenshot / snapshot / evaluate / aria_snapshot — approved by default (read-only)
  • Click / fill on sensitive targets — triggers OAuth3 approval gate
