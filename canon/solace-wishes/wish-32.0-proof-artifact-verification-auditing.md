## 4. Invariants (7 Total)

**INV-1:** Proof JSON must contain all required fields
- Enforced by: Schema validation against proof-v1.schema.json
- Required: spec_id, timestamp, recipe_id, execution_id, signature_god_65537, execution_trace, screenshots
- Fail mode: Test FAILS if any required field missing

**INV-2:** Cryptographic signature must be valid
- Enforced by: Verify signature = HMAC-SHA256(content, secret_65537)
- Fail mode: Test FAILS if signature doesn't match

**INV-3:** Hash chain must be unbroken
- Enforced by: hash(recipe_id) → hash(execution_trace) → hash(proof) all match
- Fail mode: Test FAILS if any hash doesn't match in chain

**INV-4:** Execution timeline must be chronologically ordered
- Enforced by: `timestamp[i] <= timestamp[i+1]` for all actions
- Fail mode: Test FAILS if any timestamp out of order

**INV-5:** Action sequence must match recipe specification
- Enforced by: For each action: action.id, action.type, action.target all exist and match recipe
- Fail mode: Test FAILS if action doesn't match recipe

**INV-6:** All screenshots must have matching hashes
- Enforced by: SHA256(screenshot_file) == proof.screenshot_hash
- Fail mode: Test FAILS if hash doesn't match

**INV-7:** Proof must be unmodified since signing
- Enforced by: Signature verification (any modification invalidates signature)
- Fail mode: Test FAILS if signature invalid

### T3: Hash Chain Validation (Recipe → Execution → Proof)

```
Setup:   Proof with valid signature
Input:   Reconstruct and verify complete hash chain
Expect:  All hashes in chain match
Verify:
  - From proof, extract: recipe_id, recipe_hash (if stored)
  - From proof, extract: execution_trace, compute execution_hash = SHA256(json.stringify(execution_trace))
  - From proof, extract: proof_payload, compute proof_hash = SHA256(json.stringify(proof_payload))
  - Chain verification:
    1. recipe_hash (baseline) extracted from proof metadata
    2. execution_hash = SHA256(execution_trace) ✅ matches proof.execution_trace_hash
    3. proof_hash = SHA256(proof) ✅ matches proof.proof_hash
  - Chain order: recipe → execution → proof (each depends on previous)

Hash Values (Example):
  - recipe_hash: abc123def456ghi789jkl012mno345pqr
  - execution_trace_hash: def456ghi789jkl012mno345pqr678stu
  - proof_hash: ghi789jkl012mno345pqr678stu901vwx

Harsh QA:
  - If any hash missing: FAIL
  - If execution_hash doesn't match proof.execution_trace_hash: FAIL (trace modified)
  - If proof_hash doesn't match expected: FAIL (proof modified)
  - If chain order wrong: FAIL
```

