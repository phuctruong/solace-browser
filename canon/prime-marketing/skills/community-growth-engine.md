## GENOME-79: THE 5 GROWTH ENGINES

### Engine 1: ACP Framework (Audience → Community → Product)

Source: Greg Isenberg

```
TRADITIONAL (broken):  Build Product → Find Customers → Hope
ACP (works):           Build Audience → Build Community → Extract Product

PHASE 1 — AUDIENCE (Days 1-90):
  1. Identify target niche (specific, passionate, underserved)
  2. Find your format (test one format daily for 30 days)
  3. Build content system (idea capture → creation → distribution)
  4. Set ONE metric (not vanity — choose: email subscribers, followers, or engagement)
  5. Evaluate at 90 days with data, not gut

PHASE 2 — COMMUNITY (Days 91-180):
  Platform: Discord / Slack / GitHub Discussions
  Structure:
    - Welcome channel with onboarding flow
    - Help channel (members help members)
    - Showcase channel (member projects)
    - Announcements (1-2/week max)
  Cadence:
    - Weekly AMA or office hours
    - Monthly demo day
    - Quarterly launch week

PHASE 3 — PRODUCT (Days 181+):
  Extract from community:
    - What do members ask for most? (Counter() the requests)
    - What do members build with your tools?
    - What would members PAY for?
  Build the product the community already wants.
```

### Engine 2: Launch Week Cadence

Source: Supabase (1M → 4.5M developers in <1 year, near-zero ad spend)

```
LAUNCH_WEEK:
  Duration: 5 days (Mon-Fri)
  Cadence: Every 3-4 months (quarterly)
  Structure: 1 major feature announcement per day

  Day 1: Biggest announcement (maximize Monday attention)
  Day 2: Developer-facing feature (technical depth)
  Day 3: Community feature (integration, plugin, template)
  Day 4: Design/UX update (visual, shareable)
  Day 5: Future roadmap + community celebration

  RULES:
    - Each day has dedicated blog post + social thread
    - Each announcement is independently shareable
    - Build anticipation 2 weeks before (countdown)
    - Collect all launches on a single "Launch Week" page
    - Post-mortem after each week (what worked, what didn't)

  WHY IT WORKS:
    - Concentrated buzz > distributed noise
    - 5 chances to reach any given person
    - Creates FOMO and anticipation cycle
    - Forces shipping cadence
```

### Engine 3: Ecosystem Flywheel

Source: Lenny Rachitsky

```
FLYWHEEL:
  Newsletter → Paid Tier → Community → UGC → Podcast → Newsletter

  1. Newsletter builds reach + trust (free, consistent, valuable)
  2. Paid conversion drives community access (premium tier)
  3. Community engagement creates retention moat (members help members)
  4. Community UGC fuels content (testimonials, case studies, tutorials)
  5. Content reach feeds newsletter with new subscribers
  6. Ecosystem scale enables new products (job board, courses, marketplace)

  THREE PRINCIPLES:
    - "Give before you ask" (9 months free before monetization)
    - "Design for compounding" (each layer strengthens others)
    - "Monetize momentum" (layer value strategically)
```

### Engine 4: Zero-Click Content Strategy

Source: Rand Fishkin (SparkToro), Amanda Natividad

```
ZERO_CLICK:
  60% of Google searches end without a click
  97.3% of Facebook post views exclude external links
  LinkedIn non-link posts get 8x reach vs link posts

BANKING SYSTEM (4-5:1 ratio):
  - Publish 4-5 posts with embedded value (NO links)
  - Every 5th post can include a CTA
  - Build algorithmic capital before spending it

PLATFORM RULES:
  Twitter/X:  Standalone threads with insights. Link in reply only.
  LinkedIn:   Lead with insight. Link in comment only. 600-1200 words.
  Reddit:     10% max self-mention. Community participation first.
  HackerNews: Technical depth. Quality over quantity. 2 posts/week max.

MEASUREMENT (VOI not ROI):
  - Branded search lift over time
  - Audience growth (followers, engagement, shares)
  - Content sustainability (evergreen vs ephemeral)
  - NOT: click-through rate (irrelevant in zero-click world)
```

### Engine 6: 90-9-1 Developer Community

Source: Decibel VC Developer Marketing Playbook

```
90-9-1 RULE:
  90% = Lurkers (consume content silently)
  9%  = Contributors (ask questions, file issues)
  1%  = Advocates (create content, evangelize, contribute code)

STRATEGY:
  For the 90%: Create excellent docs, tutorials, blog posts
  For the 9%:  Treat support questions as research (voice of customer)
  For the 1%:  Celebrate publicly (contributor spotlights, advisory roles)

GOAL: Convert 9% → 1% through "strategic love, care, and attention"

ANTI-PATTERNS:
  ❌ Don't hire non-technical community managers
  ❌ Don't staff events without people who can answer technical questions
  ❌ Don't treat community as marketing channel (it's a product)
  ❌ Don't conflate marketing with paid advertising for developers
```

## STATE MACHINE

```
STATES = {
  AUDIENCE_BUILD,      # Content creation, format testing
  PLATFORM_SETUP,      # Discord/Slack/GitHub Discussions
  SEED_COMMUNITY,      # First 100 members (personal invites)
  ENGAGEMENT_CADENCE,  # Weekly rhythm established
  CONTENT_FLYWHEEL,    # UGC + newsletter + social loop
  LAUNCH_WEEK,         # Quarterly concentrated launches
  MONETIZE,            # Premium tier, paid features
  SCALE                # Self-sustaining growth
}

TRANSITIONS:
  AUDIENCE_BUILD → PLATFORM_SETUP      (≥500 followers/subscribers)
  PLATFORM_SETUP → SEED_COMMUNITY      (platform configured)
  SEED_COMMUNITY → ENGAGEMENT_CADENCE  (≥100 members)
  ENGAGEMENT_CADENCE → CONTENT_FLYWHEEL (DAU/MAU ≥ 10%)
  CONTENT_FLYWHEEL → LAUNCH_WEEK       (content pipeline stable)
  LAUNCH_WEEK → MONETIZE               (≥1000 community members)
  MONETIZE → SCALE                     (positive unit economics)

FORBIDDEN:
  MONETIZE before ENGAGEMENT_CADENCE (extracting before giving)
  LAUNCH_WEEK before CONTENT_FLYWHEEL (events without content = empty)
  SCALE before SEED_COMMUNITY (premature scaling = death)
```
