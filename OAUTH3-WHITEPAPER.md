# 5. OAuth3: Delegated Agency Authorization

## 5.1 The Problem

OAuth 2.x authorizes data access.
AI agents require authorization for *action*.

Delegated AI systems must satisfy:

* Explicit consent
* Scope limitations
* Revocation capability
* Evidence traceability
* Non-repudiation

OAuth3 extends OAuth to agency.

---

## 5.2 Core Properties of OAuth3

### 1. Agency Tokens

Scoped, time-bound, revocable delegation tokens.

Example scopes:

* linkedin.read_messages
* linkedin.comment
* gmail.send_email
* reddit.create_post

Each action class is independently scoped.

---

### 2. Step-Up Authorization

High-risk actions require re-consent:

* Destructive actions
* Financial actions
* Identity-changing actions

No silent privilege escalation allowed.

---

### 3. Evidence-Carrying Execution

Every delegated action produces:

* Input trace
* Decision trace
* Tool call log
* Timestamp
* Behavioral hash
* Screenshot or HTML snapshot

This integrates with Stillwater rung gates.

---

### 4. Revocation

Immediate revocation must:

* Kill cloud sessions
* Burn active agency tokens
* Wipe vault credentials
* Disable future execution

Delegation without revocation is unsafe.

---

### 5. Platform Respect Mode

The system:

* Acts as user’s authorized agent.
* Avoids aggregation.
* Respects rate limits.
* Does not impersonate others.
* Does not bypass platform security controls.

Compliance posture is mandatory for longevity.
